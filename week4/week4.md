## 1. 뷰 바인딩

뷰 바인딩은 XML 파일에 선언한 뷰 객체를 코드에서 쉽게 이용하는 방법입니다.

XML 파일에 등록한 뷰는 "@+id/ex1"  id 속성을 추가합니다.
R.class 안에 존재하는 정적 중첩 클래스 id 클래스에 정적 변수 ex1을 추가합니다.
추가된 정적 변수 ex1을 다른 코드에서 가져와 사용하기 위해 **findVeiwById()** 를 사용합니다.


그런데 뷰 바인딩은 findViewById() 함수를 이용하지 않고 XML파일에 등록된 뷰 객체를 쉽게 사용하는 기술입니다.

**뷰 바인딩을 사용하려면 그래들 파일(build.gradle)에 다음처럼 선언해야 합니다.**

```build.gradle

android {
   (... 생략함 ...)
    viewBinding.isEnabled = true  //핵심 추가
}

```

저렇게 설정하면 **레이아웃 XML 파일에 등록된 뷰 객체를 포함하는 클래스가 자동으로 만들어집니다.**

즉, findViewById()를 사용하지 않고 자동으로 만들어진 클래스를 이용해 뷰에 접근하고 사용하면 됩니다.

자동으로 만들어지는 클래스의 이름은 레이아웃 XML 파일명을 따릅니다.

첫 글자를 대문자로 하고 밑줄(_)은 빼고 뒤에 오는 단어를 대문자로 만든 후 파일명 마지막에 **Binding**을 추가합니다.  (카멜케이스 스타일)

#### ex)  XML 파일명 : item_main.xml  ->  자동으로 만들어진 클래스 이름 : ItemMainBinding.class
